# Parameters (파라미터)

> 모델이 학습을 통해 스스로 조정하는 값
> 학습 전엔 랜덤값, 학습 후엔 데이터를 잘 설명하는 값

---

## 직관적으로 이해하기

선형 회귀에서:

```
y = weight × x + bias

처음:    weight = 랜덤값, bias = 랜덤값
학습 후: weight ≈ 2.0,   bias ≈ 1.0
```

weight와 bias가 파라미터.  
역전파를 통해 점점 정답에 가까운 값으로 업데이트됨.

---

## 신경망에서의 파라미터

```
nn.Linear(4, 64)

weight: 4 × 64 = 256개  ← 입력과 출력을 연결하는 가중치
bias:        64개        ← 각 출력 뉴런의 편향

합계: 320개
```

모든 연결마다 weight가 하나씩 존재:

```
입력 뉴런 4개 × 출력 뉴런 64개 = 256개의 연결
                                = 256개의 weight
```

---

## 학습이란 결국

```
학습 전:
  파라미터 = 랜덤값 (아무것도 모르는 상태)

학습 중:
  데이터를 보고 Loss 계산
  역전파로 기울기 계산
  옵티마이저가 파라미터 업데이트
  → 반복

학습 후:
  파라미터 = 데이터를 잘 설명하는 값
```

파라미터를 잘 찾는 과정이 곧 학습.

---

## 파라미터 수 계산

```python
total = sum(p.numel() for p in model.parameters())
# p.numel(): 텐서의 전체 원소 수
```

### BC 모델 (4,610개)

| 레이어 | weight | bias | 합계 |
|--------|--------|------|------|
| Linear(4→64) | 4×64 = 256 | 64 | 320 |
| Linear(64→64) | 64×64 = 4,096 | 64 | 4,160 |
| Linear(64→2) | 64×2 = 128 | 2 | 130 |
| **전체** | | | **4,610** |

### MNIST 모델 (109,386개)

| 레이어 | weight | bias | 합계 |
|--------|--------|------|------|
| Linear(784→128) | 784×128 = 100,352 | 128 | 100,480 |
| Linear(128→64) | 128×64 = 8,192 | 64 | 8,256 |
| Linear(64→10) | 64×10 = 640 | 10 | 650 |
| **전체** | | | **109,386** |

BC 모델이 훨씬 적은 이유: 입력이 784 → 4로 줄었기 때문.

## 파라미터 vs 하이퍼파라미터

헷갈리기 쉬운 개념:

| | 파라미터 | 하이퍼파라미터 |
|---|---|---|
| 예시 | weight, bias | lr, batch_size, epochs |
| 결정 방식 | 학습으로 자동 조정 | 사람이 직접 설정 |
| 저장 여부 | 학습 후 저장됨 | 코드에 고정 |
