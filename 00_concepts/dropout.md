# Dropout

> 학습 중 랜덤으로 일부 뉴런을 꺼버려 과적합을 방지하는 기법

---

## 왜 필요한가 — 과적합 (Overfitting)

```
과적합: 학습 데이터는 완벽하게 맞히는데
        새로운 데이터는 못 맞히는 현상

예:
  학습 데이터 정확도: 99%
  테스트 데이터 정확도: 75%  ← 문제!
```

시험 문제를 답까지 통째로 외운 학생처럼, 모델이 패턴을 이해한 게 아니라 그냥 외운 것.

---

## 동작 방식

```python
nn.Dropout(p=0.5)  # 50% 확률로 뉴런을 랜덤하게 끔
```

```
학습 중:
  뉴런 [1, 2, 3, 4, 5, 6]
       ↓ 50% 랜덤으로 끔 (매 배치마다 다르게)
  뉴런 [1, 0, 3, 0, 5, 0]

평가 중 (model.eval()):
  모든 뉴런 활성화 → Dropout 비활성화
```

---

## 왜 효과가 있나

특정 뉴런에 의존하지 못하게 만들어 더 robust한 학습 유도.

```
Dropout 없음: "이 패턴은 뉴런 3번이 항상 처리해"
               → 뉴런 3번이 특정 데이터만 외움

Dropout 있음: "뉴런 3번이 꺼질 수도 있으니
               여러 뉴런이 함께 처리해야 해"
               → 더 일반화된 학습
```

---

## 사용 방법

```python
self.model = nn.Sequential(
    nn.Flatten(),
    nn.Linear(784, 128),
    nn.ReLU(),
    nn.Dropout(p=0.3),   # 30% 뉴런 끔
    nn.Linear(128, 64),
    nn.ReLU(),
    nn.Dropout(p=0.3),
    nn.Linear(64, 10)
)
```

일반적으로 p=0.2~0.5 사이 값을 사용.  
너무 크면 (p=0.8 이상) 오히려 학습이 잘 안 됨.

---

## model.train() vs model.eval()과의 관계

```
model.train() → Dropout 활성화   (학습 중)
model.eval()  → Dropout 비활성화 (평가 중)
```

평가할 때 반드시 `model.eval()`을 써야 하는 이유가 바로 Dropout 때문.  
끄지 않으면 평가 중에도 뉴런이 꺼져서 정확도가 낮게 나옴.

---

## 언제 쓸까

| 상황 | Dropout 필요 여부 |
|------|------------------|
| 모델이 작고 데이터가 많음 | 불필요할 수 있음 |
| 모델이 크고 데이터가 적음 | 필수 |
| 학습/테스트 정확도 차이가 클 때 | 추가 권장 |
